# Dashboards

Generated by `network-chief dashboard`. Two kinds of files live here:

| File | Source | Refresh |
|---|---|---|
| `example.md` | A representative dashboard generated from the smoke-test sample data in `examples/`. Refreshed on every push by `.github/workflows/dashboard-example.yml`. | automatic |
| `dashboard-<window>d.md` (when present) | Your real network dashboard, written by `network-chief dashboard --window <N>`. | manual or via OpenClaw cron |

The five-pillar shape and KPI provenance are documented inline in each file under "How to read this." Highlights:

- **Stale-but-valuable** is the single most actionable number — high-value contacts who have not been touched in 90 days.
- **Approval rate** is a calibration metric — if it drifts below 50% over consecutive snapshots, the agent's drafts are diverging from your judgment and the value heuristics or goals need tuning.
- **Reciprocity (incoming÷outgoing 90d)** flags broadcast-heavy or receive-heavy patterns. Healthy operators run roughly 0.7–1.3.

## Privacy

Real network data is sensitive even at the aggregate level. If you make this repo public, do **not** commit `dashboard-<window>d.md` — only the smoke-driven `example.md` is safe to publish. The repo's `.gitignore` does not exclude `dashboards/` automatically, so pushing or omitting your real dashboards is your conscious choice.

To redact your live dashboard from the repo without losing local copies:

```bash
git rm --cached dashboards/dashboard-*.md dashboards/dashboard-*.json
echo "dashboards/dashboard-*.md" >> .gitignore
echo "dashboards/dashboard-*.json" >> .gitignore
git commit -am "Stop publishing live dashboards"
```
